# 仲裁智能化项目 v1 UI 规范

## 1. 总体风格

- 产品类型：仲裁委 B 端工作台
- 目标气质：专业、可信、克制、清晰
- 关键词：浅底分层、权威蓝、真实业务感、结构优先
- 禁止方向：泛 AI 风格、黑底科技感大屏、抽象炫光界面、脱离仲裁委语境的互联网后台
- 不依赖 Element Plus，使用自建纯 CSS 设计系统

## 2. 色彩策略

- 主品牌色：深权威蓝（#2f5cf5）
- 主品牌色浅色：浅蓝高亮（#ebf2ff）
- 页面背景色：浅灰蓝近白色（#f3f6fb）
- 卡片背景色：纯白
- 边框色：冷灰蓝边框（#dce5f2）
- 文本主色：深灰蓝（#344054）
- 文本次色：中性灰蓝（#6b778c）
- 成功色：绿色（#10B981 / #1f9d66）
- 预警色：橙色（#F59E0B / #d9902f）
- 风险色：红色（#EF4444 / #e25555）

规则：
- 大面积背景保持浅色
- 主色用于标题头、主按钮、关键状态
- 风险色只服务业务状态

## 3. 字体与排版

- 正文字体：系统无衬线
- 标题层级控制在 3 级以内
- 字重以 600 / 700 为主
- 文书编辑器内文本 line-height: 1.8，标题居中，段首缩进 2em

## 4. 布局原则

- 默认工作台全宽三栏布局
- 编辑器中区使用白纸样式（.wb-paper：max-width 800px、居中、白底、圆角、阴影）
- 首屏必须回答"这个案子在争什么"
- 高密度信息通过分组和层次解决

## 5. 组件规范

### 按钮
- 主按钮：蓝色，数量严格控制
- 次按钮：白底描边
- 危险按钮：只用于高风险动作

### 卡片
- 默认卡片：白底、浅边框（.wb-card）
- 高亮卡片：边框变色 + 浅底强调
- 风险卡片：浅红底或左色条

### 浮动工具栏
- 选中文本后出现在选区上方
- 白底圆角阴影，图标+文字的横排按钮
- 包含 5 个 AI 操作（续写/改写/扩写/精简/总结）和批注

### 批注（飞书文档风格）
- 从选中文本发起
- 有批注的段落在文中持久显示浅黄底 + 右侧橙色竖线，hover 加深
- 点击文中批注段落 → 右侧切到人工批注 Tab，对应卡片闪烁高亮
- 点击批注卡片 → 文中段落 Overlay 高亮 5 秒
- 卡片顶部展示选中原文引用（黄色竖线+文本）
- hover 显示操作图标（编辑/已解决/删除）
- 状态：待处理（黄点）/ 已解决（绿点）

### 手风琴式材料面板（文书写作助手左栏）
- 每个材料项点击展开/收起其下证据条目
- 材料标题行：文件名完整显示（不截断），无引用条数/已引用标签
- 顶部提交方筛选（全部/申请方/被申请方/仲裁庭），筛选按钮单行横向排布不换行
- 证据卡片无右上角标签（通过/高风险）
- 点击「引用」在光标处直接插入，插入后 Overlay 高亮 5 秒
- 材料标题行提供「预览」按钮，点击打开文件查看器弹窗（模拟 PDF 编辑器）

### 模块导航条（编辑器顶部）
- 工具栏下方横向标签：首部·仲裁请求·案件事实·答辩·查明·认为·裁决主文
- 点击跳转，滚动跟随自动高亮当前模块（navScrollLock 防止误切）
- 当前模块蓝底白字，其他模块灰色
- 文书写作助手与文书审核统一复用同一导航结构

### 文件预览弹窗（文书写作助手 / 文书审核）
- 弹窗结构：标题区（文件名+格式+大小）→ 工具条（上一页/下一页/缩放）→ 纸张内容区
- 背景使用浅灰蓝遮罩，纸张区域白底并有轻阴影
- 内容区默认展示“文件摘要 + 关键审查提示”，支持 `white-space: pre-wrap`
- 关闭按钮固定在右上角，支持遮罩点击关闭

### 庭审记录提取模块（文书审核左栏）
- 固定三项按钮：核心诉求、无争议事实认定、核心争议焦点分析
- 三个按钮使用不同深浅蓝 icon，保持同一视觉家族
- 点击按钮打开对应结构化弹窗，不做正文定位跳转

### 校对检查与定位高亮
- 有问题的段落左侧显示 3px 竖线（红色=error，橙色=warning）
- 点击校对卡片 → 文中对应段落 Overlay 红/橙色高亮 5.5 秒
- 点击批注卡片 → 文中对应段落 Overlay 黄色高亮 5 秒
- 高亮采用 Overlay 覆盖层实现，不修改 ProseMirror DOM

### Toast 消息
- 全局顶部居中浮出
- 类型：success（绿）/ error（红）/ warning（橙）/ info（蓝）
- 2.5 秒自动消失
- 定位失败时显示「未找到对应段落，请确保文档已加载」

### 自动保存指示器
- 编辑器工具栏右侧
- 绿色文字 + 勾选图标 + "已自动保存 HH:MM"

## 6. 状态表达

- 成功：绿色标签或浅绿边框
- 关注：橙色标签或浅橙边框
- 风险：红色标签或浅红提示
- 缺失：灰红色提示
- 禁用：灰色文本和灰底

## 7. 动效原则

- 只做进入、切换、定位、高亮反馈
- 校对/批注定位使用 scrollIntoView + Overlay 高亮 5–5.5 秒
- 手风琴展开使用 slideDown
- Toast 使用 slide + fade 进出
